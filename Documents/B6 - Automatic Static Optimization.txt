## Automatic Static Optimization
- Ở NextJs họ sẽ xác định là ở 1 cái trang nó có static hay không, tức là nó có thể pre-render được hay không thì nó sẽ xác định bằng cách là: mình có sử dụng cái hàm getServerSideProps hoặc là cái hàm getInitialProps hay không. Nếu như mình có sử dụng thì tức là nó là những server side, nó là như kiểu mình sẽ request theo từng cái request, mình sẽ render theo từng cái request cho nên không thể pre-render trước đó được. Cho nên nếu mình sử dụng 2 hàm này thì nó không phải là Automatic Static Optimization, còn nếu như mình không sử dụng 2 cái hàm này thì tức là cái trang đó nó pre-render được thì nó sẽ bật cái mode Automatic Static Optimization. Thì khi mà cái trang của mình mà có thể pre-render được như thế này thì nó sẽ ouput 1 cái file HTML khi mà mình build (lúc mà run cái lệnh next build ấy), còn những cái trang sử dụng getServerSideProps thì nó sẽ output ra 1 file JS, như vậy rõ ràng nếu như mình mà tạo ra 1 file HTML thì lúc mình request lên nó sẽ trả file HTML về, còn nếu nó đưa ra 1 cái file JS thì nó phải load cái js đó về sau đó nó phải chạy cái file đó thì nó mới ra được cái markup.
** Lưu ý khi mà sử dụng router.query thì đối với những cái thằng ASO thì lần đầu tiên cái object query thì nó sẽ empty nhưng mà sau khi nó hydration xong thì nó sẽ update trigger 1 lần update nữa nếu như cái object query đó có sự thay đổi, còn với thằng Not ASO thì lúc nào cũng có (hiểu hơn xem code và để ý)
